# 测试需求：微闪贴预警客户交易信息查询（多接口）

## 场景描述
查询客户号为 962020122711000002 的微闪贴预警交易信息，模型编号为 WSTY001。

## 接口信息
- 接口1（交易信息）：/aml/wst/custTransInfo
查询预警日期 <= 案例日期下的最新预警日期的交易
- 接口2（背书转让记录）：/aml/wst/custDrftRecord
查询预警日期 <= 案例日期下的最新预警日期的背书转让记录

## 业务约束
1. 同一个客户有预警必须有交易（1:N）
2. 同一个客户有案例必须有预警（1:N）

## 约束条件
- cust_id 固定为 962020122711000002
- model_key 固定为 WSTY001

## 场景生成约束
1. 测试场景必须覆盖时间条件相关的边界情况，而不仅是单一路径的命中数据。
2. 对“预警日期 <= 案例日期并取最近值”的业务语义，需构造候选日期不唯一的数据，验证排序后取值是否正确。
3. 每个核心业务功能至少覆盖三类数据分支：满足条件数据、边界临界数据、不满足条件/无命中数据。
4. 两个接口需基于同一客户与模型约束，校验时间筛选与取值规则在跨接口上的一致性。

## 验收标准
- 场景中应出现用于验证“最近值选择”正确性的时间边界样本，且样本可区分“被选中值”和“未被选中值”。
- SQL 造数结果中，关键时间字段应具备可用于边界验证的多样取值，不应全部为同一日期。
